Auto-open any new or modified code file (including .css, .json like package.json and composer.json) or README file when the change comes from Cursor agent.
Do not open files generated by Composer, inside vendor/, from git operations, or non-code files except README (.md, .mdx, .mdc). Lock files (yarn.lock, package-lock.json) remain excluded.

**Laravel:** Never open files in `storage/` directory (especially `storage/framework/views/` compiled Blade templates). Only open source files in `resources/views/`. Browser refreshes must not trigger file opening.

**Package Managers & Build Scripts:** Aggressively detect when Composer, npm, yarn, or build tools are running by monitoring early indicators:
- `package.json`, `composer.json` (read at operation start)
- Lock files (`composer.lock`, `package-lock.json`, `yarn.lock`) - change early
- `vendor/autoload.php`, `vendor/composer/installed.json` (composer early files)
- `node_modules/.package-lock.json` (npm metadata)
- Build output directories and manifests

Suppress all auto-open operations for 30 seconds (extended with each activity) to avoid opening framework-generated, installed, or compiled files. Early detection ensures suppression activates BEFORE files are created.
